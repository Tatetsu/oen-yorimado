# プロジェクトルール

## 基本情報
- **プロジェクト名**: 児童施設 来館記録管理システム（oen-yorimado）
- **技術スタック**: Google Sheets + Google Apps Script + Google Forms
- **設計書**: docs/ 配下を参照

## 役割と目的
- あなたの役割は、児童施設の来館記録管理システムの設計・GAS実装を支援することです。
- 常にプロジェクトの目的（docs/requirements.md）を念頭に置いて回答してください。

## プロジェクト固有ルール

### スプレッドシート設計方針
- スプレッドシート内の関数は**最小限**に抑える（パフォーマンス・保守性の観点）
- 集計・表示更新はGASで「値」として書き込む方式を採用する
- ドロップダウンの選択肢は児童マスタから直接GASで取得する（プルダウンマスタシートは不要）

### GAS実装方針
- 1ファイル1責務（setup.gs, allocation.gs 等）
- 共通処理は utils.gs に集約する
- シート名・列番号はマジックナンバーを避け、定数として定義する
- エラー発生時はログに記録し、処理を中断しない（可能な限り）

### データの扱い
- フォームの回答シートは手動編集不可（GASからも書き込まない）
- 振り分け記録は実データと明確に区別する（データ区分列で管理）
- 確定来館記録 = フォームの回答 + 振り分け記録の統合
- 児童別ビューは確定来館記録のみを参照する

### 営業日の定義
- 初期設定: 全曜日が対象
- 祝日カレンダーによる除外は将来ユーザー側で作成・適用予定

### 制約
- 1日あたり最大7名
- GAS無料枠内で運用する（実行時間6分以内）
