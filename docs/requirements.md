# 要件定義書: 児童施設 来館記録管理システム

## 1. 概要

### 1.1 目的
児童施設における来館記録をGoogleスプレッドシート + GAS（Google Apps Script）で管理するシステムを構築する。スタッフがGoogleフォームで入力した来館記録を集計し、月間利用枠に対する実績管理・余りポイントの自動振り分け・児童別ビューの出力を行う。

### 1.2 設計方針
- **GAS処理ベース**: スプレッドシート内の関数は最小限に抑え、集計・表示更新はGASで「値」として書き込む
- **理由**: パフォーマンス確保、関数破損リスクの排除、メンテナンス性の向上

### 1.3 システム構成
- Google スプレッドシート（データストア + UI）
- Google Apps Script（ビジネスロジック）
- Google フォーム（スタッフ入力インターフェース）

---

## 2. シート構成

### 2.1 シート一覧

| No. | シート名 | 役割 | データ更新方法 |
|-----|---------|------|--------------|
| 1 | フォームの回答 | Googleフォームの自動反映先（実データ） | 自動（フォーム送信時） |
| 2 | 児童マスタ | 児童の基本情報管理 | 手動 |
| 3 | 月別集計 | 年月指定で全児童の来館数一覧 | GAS（自動トリガー + 手動ボタン） |
| 4 | 確定来館記録 | 実データ + 振り分けデータの統合シート（データ区分列で識別） | GAS（フォーム送信時 + 振り分け実行時） |
| 5 | 児童別ビュー | 1名の来館履歴詳細（印刷対応）。月別・年別・全期間表示に対応 | GAS（ドロップダウン選択 + ボタン） |
| 6 | 来館カレンダー | 日×児童のマトリクス形式で来館状況を一覧表示（○=実データ、△=振り分け）。月別・年別表示に対応 | GAS（手動ボタン） |
| 7 | ログ | GASエラーログの記録シート | GAS（エラー発生時に自動記録） |

### 2.2 データフロー

```
[Googleフォーム]
    ↓ 自動反映
[フォームの回答] ──→ [確定来館記録] ←── [児童マスタ]
                     (実データ+振り分け)   ↓ (振り分けロジック)
                          ↓               ↓
                     [確定来館記録]
                     (振り分けデータ直接追加)
                          ↓
                     [月別集計]
                          ↓
                     [児童別ビュー]
                     [来館カレンダー]
```

## 3. 各シート詳細仕様

| 列 | 項目 | 型 |
|----|------|-----|

| B | 記録日 | 日付 |
| C | スタッフ名 | テキスト |
| D | 児童名 | テキスト（プルダウン選択） |
| E | 入所時間 | 時刻 |
| F | 退所時間 | 時刻 |
| G | 体温 | 数値 |
| H | 食事 | ○ / × |
| I | 入浴 | ○ / × |
| J | 睡眠 | ○ / × |
| K | 便 | ○ / × |
| L | 服薬 | ○ / × |
| M | その他連絡事項 | テキスト |
| N | メール送信済 | テキスト | GASが自動書き込み。「送信済 yyyy/MM/dd HH:mm」形式 |

### 3.2 児童マスタ（既存 + 拡張）

| 列 | 項目 | 型 | 備考 |
|----|------|-----|------|
| A | No. | 数値 | 一意識別子 |
| B | 児童名 | テキスト | |
| C | 保護者名 | テキスト | |
| D | 保護者メールアドレス | テキスト | |
| E | 月間利用枠 | 数値 | |
| F | 医療型の有無 | あり / なし | |
| G | 担当スタッフ | テキスト | |
| H | 入所状況 | 稼働 / 休止 / 退所 | |
| I | 来館曜日 | テキスト | 例: 月,水,金（半角・全角カンマ、スペース、「月曜日」形式にも対応） |
| J | 優先度 | 数値 | **新規追加** 振り分け優先順位（1が最優先、数値が小さいほど優先） |

### 3.3 月別集計

年月をドロップダウンで選択し、全児童の来館実績を一覧表示する。

**操作エリア（1行目）:**

| セル | 項目 | 入力方法 |
|------|------|---------|
| B1 | 対象年月 | ドロップダウン |

**データエリア（2行目=ヘッダー、3行目〜=データ）:**

| 列 | 項目 | データソース |
|----|------|------------|
| A | No. | 児童マスタ |
| B | 児童名 | 児童マスタ |
| C | 月間利用枠 | 児童マスタ |
| D | 来館数 | 確定来館記録から集計 |
| E | 残数 | C - D |
| F | 利用率 | D / C（%表示） |

**データソース:** フォームの回答（実記録のみ）から集計。振り分けデータは含まない。

**更新タイミング:**
- フォーム送信時（自動トリガー）
- 手動ボタン押下時
- B1セル変更時（onEditトリガー）

### 3.4 確定来館記録

フォームの回答（実データ）+ 振り分けデータを統合した最終データ。
振り分け実行時に、補完値（スタッフ名・入退所時間・生活記録等）を算出してデータ区分「振り分け」として直接追加する。
**月別集計・児童別ビュー・来館カレンダーはこのシートを参照する。**

| 列 | 項目 | 型 | 備考 |
|----|------|-----|------|
| A | 記録日 | 日付 | |
| B | 児童名 | テキスト | |
| C | データ区分 | テキスト | 「実データ」/「振り分け」 |
| D | スタッフ名 | テキスト | 振り分けの場合は同月実データの最頻値（なければ児童マスタの担当スタッフ） |
| E | 入所時間 | 時刻 | 振り分けの場合は同月実データの最頻値（なければ09:00） |
| F | 退所時間 | 時刻 | 振り分けの場合は同月実データの最頻値（なければ17:00） |
| G | 体温 | 数値 | 振り分けの場合は同月実データの最頻値（なければ36.5） |
| H | 食事 | ○ / × | 振り分けの場合は同月実データの最頻値（なければ○） |
| I | 入浴 | ○ / × | 同上 |
| J | 睡眠 | ○ / × | 同上 |
| K | 便 | ○ / × | 同上 |
| L | 服薬 | ○ / × | 同上 |
| M | その他連絡事項 | テキスト | 自児童ノートからランダム選択→他児童→「特になし」 |

**補完値の算出ルール:**
- 同月の自児童の実データ（フォームの回答）から最頻値を算出
- 実データがない場合はデフォルト値を使用
- 連絡事項: 自児童のノートからランダム選択 → なければ他児童のノートから → なければ「特になし」
- **補完値は振り分け実行時に1回だけ計算・保存される（再現性保証）**

### 3.5 児童別ビュー（印刷対応）

ドロップダウンで児童名・年月を選択し、ボタン押下で表示更新する。

**操作エリア（1〜2行目）:**

| セル | 項目 | 入力方法 |
|------|------|---------|
| B1 | 児童名 | ドロップダウン（児童マスタから） |
| B2 | 対象年月 | ドロップダウン（「すべて」「YYYY年」「YYYY年M月」） |
| D1 | 表示更新ボタン | GAS実行ボタン |

**基本情報エリア（4〜8行目）:**

| 項目 | データソース |
|------|------------|
| 保護者名 | 児童マスタ |
| 担当スタッフ | 児童マスタ |
| 月間利用枠 | 児童マスタ |
| 医療型の有無 | 児童マスタ |
| 来館回数 / 残枠 / 利用率 | 確定来館記録から集計 |

**来館履歴一覧（10行目〜）:**
- 確定来館記録から該当児童・年月のデータを日付昇順で表示
- データ区分列で「実データ」「振り分け」を識別可能

**印刷設定:**
- A4横向き
- 印刷範囲: 基本情報 + 来館履歴一覧
- ヘッダー行固定

## 4. GAS機能一覧

### 4.1 初期構築（Phase 1）

| No. | 機能名 | トリガー | 処理内容 |
|-----|--------|---------|---------|
| F-01 | シート初期セットアップ | 手動実行（1回） | 月別集計・確定来館記録・来館カレンダー・児童別ビューのシート作成、ヘッダー・書式設定 |
| F-02 | 月別集計更新 | フォーム送信時（自動）+ 手動ボタン | 確定来館記録から集計し、月別集計シートに値を書き込む |
| F-03 | 確定来館記録生成 | フォーム送信時（自動）+ 手動 | フォームの回答を確定来館記録シートに書き込む（振り分け行は保持） |
| F-04 | 児童別ビュー更新 | 手動ボタン | 選択された児童・年月の確定来館記録を児童別ビューに書き込む |

### 4.2 余りポイント振り分け（Phase 2）

| No. | 機能名 | トリガー | 処理内容 |
|-----|--------|---------|---------|
| F-05 | 余りポイント自動振り分け | 月初1日（時間トリガー） | 前月の来館記録から余りポイントを算出し、振り分けロジックを実行 |
| F-06 | 振り分け手動実行 | 手動ボタン（任意） | F-05と同じロジックを手動で実行可能にする |

### 4.3 振り分けロジック詳細（F-05 / F-06）

**入力:**
- 対象月のフォームの回答（実来館データ）
- 児童マスタ（月間利用枠、来館曜日、優先度、入所状況）

**処理手順:**

```
1. 入所状況「稼働」の児童を取得
2. 各児童の実来館回数を算出
3. 残枠（月間利用枠 - 実来館回数）がある児童を抽出
4. 優先度順にソート（数値が小さい順）
5. 確定来館記録から対象月の既存振り分け行を削除（洗い替え）
6. 各児童について:
   a. 対象月の全曜日（営業日）を取得
   b. 既に来館済みの日付を除外
   c. 来館曜日に該当する未来館日を優先候補とする
   d. 候補日の中から、その日の既存来館数が7枠未満の日を抽出
   e. 全日付の来館数が均等になるよう分散配置する
   f. 来館曜日の候補で足りない場合はランダムで他の営業日に配置
   g. 振り分け結果を確定来館記録シートにデータ区分「振り分け」として直接追加
7. 月別集計を更新（F-02）
```

**制約条件:**
- 1日あたり最大7名
- 同一児童は同一日に重複しない
- 来館曜日の日を優先して振り分ける
- 全体の日別来館数ができるだけ均等になるよう分散する
- 優先度が高い（数値が小さい）児童から先に振り分ける

### 4.4 メール通知（Phase 3）

| No. | 機能名 | トリガー | 処理内容 |
|-----|--------|---------|---------|
| F-07 | 保護者向け来館報告メール | 翌朝（時間トリガー） | 前日の来館記録（フォームの回答）から児童の様子を保護者メールアドレス宛に送信。送信済フラグで重複送信を防止 |

**メール送信仕様:**
- 手動送信はHTMLダイアログで日付をカレンダーUIから選択
- フォームの回答N列に「送信済 yyyy/MM/dd HH:mm」を書き込み、重複送信を防止
- 処理結果（送信件数・スキップ件数・エラー件数）をダイアログで表示

**メール内容（想定）:**
- 宛先: 児童マスタの保護者メールアドレス
- 件名: 【施設名】○月○日 ○○ちゃんの来館記録
- 本文: 入所時間、退所時間、体温、食事・入浴・睡眠等の記録、連絡事項

## 6. 非機能要件

| 項目 | 内容 |
|------|------|
| 規模 | 児童数: 最大10名程度、1日最大7名 |
| パフォーマンス | GAS無料枠内で運用（実行時間6分以内） |
| 可用性 | Googleサービスの可用性に依存 |
| セキュリティ | スプレッドシートの共有権限で管理、個人情報（メールアドレス等）の取り扱いに注意 |
| バックアップ | Googleスプレッドシートの版管理機能で対応 |

---

## 7. 前提条件・制約

- Googleフォームと児童マスタの児童名プルダウン同期は実装済み
- フォームで入力される児童名は必ず児童マスタに存在する（プルダウン選択のため）
- 営業日 = 全曜日（初期設定）。祝日カレンダーによる除外は将来ユーザー側で作成・適用予定
- GASの実行権限はスプレッドシートのオーナーが付与する
