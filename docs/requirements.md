# 要件定義書: 児童施設 来館記録管理システム

## 1. 概要

### 1.1 目的
児童施設における来館記録をGoogleスプレッドシート + GAS（Google Apps Script）で管理するシステムを構築する。スタッフがGoogleフォームで入力した来館記録を集計し、月間利用枠に対する実績管理・余りポイントの自動振り分け・児童別ビューの出力を行う。

### 1.2 設計方針
- **GAS処理ベース**: スプレッドシート内の関数は最小限に抑え、集計・表示更新はGASで「値」として書き込む
- **理由**: パフォーマンス確保、関数破損リスクの排除、メンテナンス性の向上

### 1.3 システム構成
- Google スプレッドシート（データストア + UI）
- Google Apps Script（ビジネスロジック）
- Google フォーム（スタッフ入力インターフェース）

## 2. シート構成

### 2.1 シート一覧

| No. | シート名 | 役割 | データ更新方法 |
|-----|---------|------|--------------|
| 1 | フォームの回答 | スタッフ用_回答シート（別スプレッドシート）からIMPORTRANGEで参照。スタッフ用_回答シートにはフォーム送信が蓄積され、スタッフがWebビューから編集・削除可能 | 自動（IMPORTRANGE） |
| 2 | 児童マスタ | 児童の基本情報管理 | 手動 |
| 3 | 設定 | システム全体のデフォルト値・動作設定 | 手動 |
| 4 | 月別集計 | 年月指定で全児童の来館数一覧 | GAS（自動トリガー + 手動ボタン） |
| 5 | 確定来館記録 | 実データ + 振り分けデータの統合シート（データ区分列で識別） | GAS（フォーム送信時 + 振り分け実行時） |
| 6 | 児童別ビュー | 1名の来館履歴詳細（印刷対応）。月別・年別・全期間表示に対応 | GAS（ドロップダウン選択 + ボタン） |
| 7 | 来館カレンダー | 日×児童のマトリクス形式で来館状況を一覧表示（○=実データ、△=振り分け）。月別・年別表示に対応 | GAS（手動ボタン） |
| 8 | ログ | GASエラーログの記録シート | GAS（エラー発生時に自動記録） |

### 2.2 データフロー

```
[Googleフォーム]
    ↓ 自動反映
[スタッフ用_回答シート] ← スタッフがWebビューから編集・削除
    ↓ IMPORTRANGE
[フォームの回答] ──→ [確定来館記録] ←── [児童マスタ]
                     (実データ+振り分け)   ↓ (振り分けロジック)
                          ↓               ↓
                     [確定来館記録]
                     (振り分けデータ直接追加)
                          ↓
                     [月別集計]
                          ↓
                     [児童別ビュー]
                     [来館カレンダー]
```

### 3.4 確定来館記録

フォームの回答（実データ）+ 振り分けデータを統合した最終データ。
振り分け実行時に、補完値（スタッフ名・入退所時間・生活記録等）を算出してデータ区分「振り分け」として直接追加する。
**月別集計・児童別ビュー・来館カレンダーはこのシートを参照する。**

**操作エリア（1〜2行目）:**

| セル | 項目 | 入力方法 |
|------|------|---------|
| B1 | 対象年 | ドロップダウン（「すべて」「YYYY年」降順）。B1/B2の組合せで行の表示/非表示を切り替える |
| B2 | 対象月 | ドロップダウン（「すべて」「1月」〜「12月」） |

**データエリア（3行目=ヘッダー、4行目〜=データ）**

**デフォルト値:** トリガーで月初に前月を集計する運用に合わせ、B1=前月の年、B2=前月（例: 今日2026-04-20 → B1="2026年"、B2="3月"）

**宿泊来館の展開ルール（ユニーク宿泊キー方式）:**

新仕様ではフォームの全レコードに入所日時・退所日時を必ず両方記入する。
児童名+入所日（日付のみ、yyyy-MM-dd）を「宿泊PK」として論理1宿泊にペアリングし、滞在期間（入所日〜退所日）を各日付に展開して書き込む。
※ 同一児童が同日に2回別々の来館をすることは無い運用前提（連泊は1夜ごとに別レコード=入所日が日ごとに異なる）。
- 例: 3/1 入所 → 3/3 退所（2泊3日）= 3行（記録日: 3/1, 3/2, 3/3）
- 入所日時・退所日時は全行に同じ値を保持する
- 1宿泊で複数レコード送信されるのは従来通り（中日の様子記録のため）
- 月別集計・年間利用枠の来館数は展開後の行数（=日数）でカウントする
- 確定来館記録の最終列に「宿泊PK」（児童名|入所日 yyyy-MM-dd）を保存する。デフォルト非表示。

**月末・年またぎの宿泊カウントルール:**

| ケース | 対象月 | カウント |
|---|---|---|
| 3/31入所→4/1退所 | 3月=1, 4月=1 | 計2 |
| 4/2入所→4/3退所 | 4月=2 | 計2 |
| 4/5入所→4/7退所（2泊3日） | 4月=3 | 計3 |
| 4/30入所→5/1退所 | 4月=1, 5月=1 | 計2 |
| 4/30入所→5/2退所（2泊3日、月またぎ） | 4月=1, 5月=2 | 計3 |
| 12/31→1/1（年またぎ） | 前年12月=1, 翌年1月=1 | 計2 |

`expandStayToDates_(checkIn, checkOut)` で日付配列を生成し、各日付が対象月（または年）に含まれるかでフィルタする。

列順はフォームの回答(2列目以降)と一致。1列目「データ区分」がフォームの「タイムスタンプ」と同位置。

| 列 | 項目 | 型 | 備考 |
|----|------|-----|------|
| A | データ区分 | テキスト | 「実データ」/「振り分け」 |
| B | 記録日 | 日付 | 連泊は宿泊日ごとに展開 |
| C | スタッフ1 | テキスト | 振り分けは同月実データの最頻値（なければ児童マスタの担当スタッフ） |
| D | スタッフ2 | テキスト | 任意。満枠日は固定スタッフで補完 |
| E | 児童名 | テキスト | |
| F | 入所日時 | 日時 | ペアリング後の論理1宿泊の値。振り分けは最頻値 |
| G | 退所予定日時 | 日時 | ペアリング後の論理1宿泊の値。振り分けは最頻値 |
| H | 送迎(往) | 数値 | 記録日==入所日 → 1 |
| I | 送迎(復) | 数値 | 記録日==退所予定日 → 1 |
| J | 体温 | 数値 | 振り分けは同月実データの最頻値（なければ36.5） |
| K | 夕食 | テキスト | 振り分けは同月実データの最頻値 |
| L | 朝食 | テキスト | 同上 |
| M | 昼食 | テキスト | 同上 |
| N | 入浴 | テキスト | 同上 |
| O | 入眠時刻 | テキスト | 振り分けは加重ランダム（実データ分布に近い形で 20:30〜22:00 から選定） |
| P | 朝4時チェック | テキスト | 振り分けは加重ランダム（睡眠 / 覚醒確認後に付き添い） |
| Q | 起床時刻 | テキスト | 振り分けは加重ランダム（実データ分布に近い形で 6:00〜7:30 から選定） |
| R | 便 | テキスト | 同上 |
| S | 服薬(夜) | テキスト | 同上 |
| T | 服薬(朝) | テキスト | 同上 |
| U | その他連絡事項 | テキスト | 自児童ノートからランダム選択→他児童→「特になし」 |
| V | 連泊 | boolean | 入所日と退所日が異なる行は true |
| W | 宿泊PK | テキスト | 児童名|入所日（yyyy-MM-dd）。デフォルト非表示 |

**補完値の算出ルール:**
- 同月の自児童の実データ（フォームの回答）から最頻値を算出
- 実データがない場合はデフォルト値を使用
- 連絡事項: 自児童のノートからランダム選択 → なければ他児童のノートから → なければ「特になし」
- **補完値は振り分け実行時に1回だけ計算・保存される（再現性保証）**

### 4.3 振り分けロジック詳細（F-05 / F-06）

**入力:**
- 対象月のフォームの回答（実来館データ）
- 児童マスタ（月間利用枠、来館曜日、非来館曜日、重度支援区分、入所状況）
- 設定シート（営業日）

**処理手順:**

```
1. 振り分け対象の児童を取得（入所状況「稼働」+ 退所かつ退所状況「別施設移動無」）
2. 各児童の実来館回数を算出
3. 残枠（月間利用枠 - 実来館回数）がある児童を抽出
4. 重度支援区分順にソート（区分1が最優先、未設定は最後）
5. 確定来館記録から対象月の既存振り分け行を削除（洗い替え）
6. 各児童について:
   a. 対象月の全日付を取得
   b. 設定シートの「営業日」に含まれない曜日を除外（未設定の場合は全曜日対象）
   c. 児童マスタの「非来館曜日」に指定された曜日を除外
   d. 既に来館済みの日付を除外
   e. 1日最大来館数に達している日を除外
   f. 「来館曜日」に該当する日を優先候補とする
   g. 全日付の来館数が均等になるよう分散配置する
   h. 来館曜日の候補で足りない場合は他の営業日（非来館曜日除く）に配置
   i. 振り分け結果を確定来館記録シートにデータ区分「振り分け」として直接追加
7. 月別集計を更新（F-02）
```

**制約条件:**
- 1日あたり最大来館数（設定シートから取得、デフォルト7名）。ただし振り分けは最大6名止まり（7名目は実データ枠として確保）
- 振り分けは設定シートの「営業日」のみ対象（空欄の場合は全曜日）
- 児童マスタの「非来館曜日」は営業日より優先して除外
- 同一児童は同一日に重複しない
- 来館曜日の日を優先して振り分ける
- 全体の日別来館数ができるだけ均等になるよう分散する
- 重度支援区分が高い（区分番号が小さい）児童から先に振り分ける
- 退所状況が「別施設移動」の児童は振り分け対象外
- 退所状況が「別施設移動無」の児童は退所月の残枠を振り分ける（翌月以降は振り分けない）
- 年間利用枠が設定されている場合、当年の確定来館記録累計との差分と月間残枠のMinを振り分け上限とする
- 月間利用枠に ±1 のランダム幅を加えた値を実効月次上限とする（例: 月間枠10日→9〜11日の範囲でランダム）。年間枠の平日按分上限と min を取り、さらに年間残枠との min を取って最終的な振り分け数を決定する。再振り分け時は再ランダムになるが、既存振り分けがある場合は確認ダイアログが表示されるため問題なし

## 7. 前提条件・制約

- Googleフォームと児童マスタの児童名プルダウン同期は実装済み
- フォームで入力される児童名は必ず児童マスタに存在する（プルダウン選択のため）

- GASの実行権限はスプレッドシートのオーナーが付与する
