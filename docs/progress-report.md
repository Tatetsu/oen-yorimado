# 児童施設 来館記録管理システム — 進捗報告書

**報告日**: 2026-03-12
**プロジェクト**: oen-yorimado（児童施設 来館記録管理システム）
**技術スタック**: Google Sheets + Google Apps Script + Google Forms + CLASP

---

## 1. システム概要

児童施設における来館記録を管理するシステム。スタッフがGoogleフォームで入力した来館記録を集計し、月間利用枠に対する実績管理・余りポイントの自動振り分け・児童別ビュー出力を行う。

**設計方針**: スプレッドシート内の関数は最小限に抑え、集計・表示更新はGASで「値」として書き込む（パフォーマンス確保・関数破損リスク排除・メンテナンス性向上のため）。

**操作エリア（1行目）:**
- B1: 対象年月（ドロップダウン）

**データエリア（2行目=ヘッダー、3行目〜=データ）:**

| 列 | 項目 | データソース |
|----|------|------------|
| A | No. | 児童マスタ |
| B | 児童名 | 児童マスタ |
| C | 月間利用枠 | 児童マスタ |

**更新タイミング**: フォーム送信時（自動）+ B1セル変更時（onEdit）+ カスタムメニュー

### 2.4 確定来館記録（GAS生成）

フォームの回答（実データ）と振り分けデータを統合した最終データシート。月別集計・児童別ビュー・来館カレンダーはすべてこのシートを参照する。

列順はフォームの回答(2列目以降)と一致。1列目「データ区分」がフォームの「タイムスタンプ」と同位置。

| 列 | 項目 | 型 | 備考 |
|----|------|-----|------|
| A | データ区分 | テキスト | 「実データ」/「振り分け」 |
| B | 記録日 | 日付 | 連泊は宿泊日ごとに展開 |
| C | スタッフ1 | テキスト | 振り分け時は補完値 |
| D | スタッフ2 | テキスト | 任意。満枠日は固定スタッフで補完 |
| E | 児童名 | テキスト | |
| F | 入所日時 | 日時 | ペアリング後の論理1宿泊の値。振り分け時は補完値 |
| G | 退所予定日時 | 日時 | ペアリング後の論理1宿泊の値。振り分け時は補完値 |
| H | 送迎(往) | 数値 | 記録日==入所日 → 1 |
| I | 送迎(復) | 数値 | 記録日==退所予定日 → 1 |
| J | 体温 | 数値 | 振り分け時は補完値 |
| K | 夕食 | テキスト | 振り分け時は補完値 |
| L | 朝食 | テキスト | 振り分け時は補完値 |
| M | 昼食 | テキスト | 振り分け時は補完値 |
| N | 入浴 | テキスト | 振り分け時は補完値 |
| O | 入眠時刻 | テキスト | 振り分け時は加重ランダム値（実データ分布から抽選） |
| P | 朝4時チェック | テキスト | 振り分け時は加重ランダム値（実データ分布から抽選） |
| Q | 起床時刻 | テキスト | 振り分け時は加重ランダム値（実データ分布から抽選） |
| R | 便 | テキスト | 振り分け時は補完値 |
| S | 服薬(夜) | テキスト | 振り分け時は補完値 |
| T | 服薬(朝) | テキスト | 振り分け時は補完値 |
| U | その他連絡事項 | テキスト | 振り分け時は補完値 |
| V | 連泊 | boolean | 連泊1宿泊から展開された行はtrue |
| W | 宿泊PK | テキスト | 児童名|入所日（yyyy-MM-dd）。デフォルト非表示 |

**振り分け時の補完値ルール:**
- 同月の自児童の実データから最頻値を算出
- 実データがない場合はデフォルト値（例: 体温36.5、入所09:00、退所17:00）
- 連絡事項: 自児童ノート → 他児童ノート → 「特になし」の順
- 補完値は振り分け実行時に1回だけ計算・保存（再現性保証）

### 2.5 児童別ビュー（GAS生成・印刷対応）

1名の来館履歴を詳細表示するシート。印刷して保護者への報告書としても使用可能。

**操作エリア（1〜2行目）:**
- B1: 児童名（ドロップダウン・児童マスタ連動）
- B2: 対象年月（ドロップダウン）
- D1: 表示更新ボタン（GAS実行）

**基本情報エリア（4〜8行目）:**
- 保護者名、担当スタッフ、月間利用枠、医療型の有無
- 来館回数 / 残枠 / 利用率

**来館履歴一覧（10行目〜）:**
- 確定来館記録から該当児童・年月のデータを日付昇順で表示
- データ区分列で「実データ」「振り分け」を識別可能

**印刷設定**: A4横向き、ヘッダー行固定

### 2.6 来館カレンダー（GAS生成）

日×児童のマトリクス形式で来館状況を一覧表示するシート。

- ○ = 実データ（フォーム入力分）
- △ = 振り分け
- 手動ボタンで更新

## 5. 振り分けロジック概要

余りポイント振り分け（F-05 / F-06）の処理フロー:

1. 入所状況「稼働」の児童を取得
2. 各児童の実来館回数を算出
3. 残枠（月間利用枠 − 実来館回数）がある児童を抽出
4. 優先度順にソート（数値が小さい順 = 高優先度から）
5. 確定来館記録から対象月の既存振り分け行を削除（洗い替え）
6. 各児童について振り分け日を決定:
   - 来館曜日に該当する未来館日を優先候補とする
   - 1日あたり最大7名の制約を適用
   - 全日付の来館数が均等になるよう分散配置
   - 来館曜日の候補で足りない場合は他の営業日に配置
7. 補完データ（スタッフ名・生活記録等）を算出し、確定来館記録に書き込み
8. 月別集計を更新

**制約条件:**
- 1日あたり最大7名
- 同一児童は同一日に重複しない
- 来館曜日の日を優先して振り分け
- 日別来館数をできるだけ均等に分散
- 優先度が高い児童から先に振り分け

## 9. 技術的な補足

- **開発環境**: CLASP（Command Line Apps Script Projects）を使用。ローカルで編集し `clasp push` でデプロイ
- **設定ファイル**: `.clasp.json` でscriptIdとrootDir（./gas）を設定済み
- **非機能要件**:
  - 児童数: 最大10名程度、1日最大7名
  - GAS無料枠内で運用（実行時間6分以内）
  - セキュリティはスプレッドシートの共有権限で管理
  - バックアップはGoogleスプレッドシートの版管理機能で対応

## 10. 前提条件・制約

- Googleフォームと児童マスタの児童名プルダウン同期は実装済み
- フォームで入力される児童名は必ず児童マスタに存在する（プルダウン選択のため）
- 営業日 = 現状は全曜日（祝日除外は未実装・将来対応予定）
- GASの実行権限はスプレッドシートのオーナーが付与
