# 00. フォームを入力する

## このページでやること

スタッフがGoogleフォームから来館記録を送信するときの**入力ルール**をまとめています。
特に **連泊（お泊り）** のときは入所日時・退所日時の入れ方が日ごとに変わるので、必ずこのページのルールに従ってください。

- **いつやるか**：来館記録を送信するとき（基本は翌朝）
- **かかる時間**：1件あたり1〜2分
- **誰がやるか**：その夜・その日の担当スタッフ

---

## 基本ルール（すべての回答に共通）

| 項目 | ルール |
|---|---|
| スタッフ1 | 必須。担当者本人を選ぶ |
| スタッフ2 | 任意。複数担当のときだけ選ぶ |
| 児童名 | プルダウンから選ぶ（手入力禁止） |
| 体温・食事・服薬・入浴・便 | 該当する時間帯のものだけ入力。なければ「-」 |
| 入眠時刻・朝4時チェック・起床時刻 | **必須**。プルダウンから選択（入眠時刻=20:30〜22:00 / 10分刻み、朝4時チェック=睡眠 or 覚醒確認後に付き添い、起床時刻=6:00〜7:30 / 10分刻み） |
| その他連絡事項 | 自由記入。引き継ぎや特記事項を記載 |

> **「記録日」項目は廃止されました（2026-04 後半）**：フォーム送信時の**送信日（タイムスタンプの日付）**が「利用日」として扱われます。**翌朝に送信** すれば前夜の利用日として正しく扱われ、メール通知も毎朝8時にその朝の送信分が拾われます。
>
> **送信タイミング**：原則「翌朝」。連泊中も**毎朝1件ずつ**送信してください（毎朝のメール通知は「過去24時間以内に送信された未送信のフォーム回答」を対象に発行されます）。

## パターン別の入力ルール

来館の形態によって、**入所日時・退所予定日時・連泊チェック**の入れ方が変わります。

### パターン①：単泊（その日のうちに帰る／1泊だけ）

通常の来館・1泊だけのお泊りはこちら。

| 項目 | 値 |
|---|---|
| 入所日時 | **入力する**（来館した日時） |
| 退所予定日時 | **入力する**（帰った／帰る予定の日時） |
| 連泊チェック | **OFF**（チェックしない） |

![単泊の入力例](images/00_form_single.png)

---

### パターン②：連泊・初日（お泊り1日目）

2泊以上のお泊りで、来館した最初の日。

| 項目 | 値 |
|---|---|
| 入所日時 | **入力する**（来館した日時） |
| 退所予定日時 | **空欄のまま**（最終日に入力するため） |
| 連泊チェック | **ON**（チェックを入れる） |

![連泊初日の入力例](images/00_form_overnight_first.png)

> **ポイント**：退所予定日時を入れてしまうと、システムが「単泊」と判定して連泊計算が崩れます。**必ず空欄**にしてください。

---

### パターン③：連泊・中日（途中の日）

3泊以上のお泊りで、初日でも最終日でもない日。

| 項目 | 値 |
|---|---|
| 入所日時 | **空欄のまま** |
| 退所予定日時 | **空欄のまま** |
| 連泊チェック | **ON**（チェックを入れる） |

![連泊中日の入力例](images/00_form_overnight_middle.png)

> **ポイント**：体温・食事・服薬・入浴・入眠時刻・朝4時チェック・起床時刻・便など**その日の様子**は通常どおり入力してください。空欄になるのは入退所日時だけです。

---

### パターン④：連泊・最終日（お帰りの日）

連泊の最後の日（お迎え／退所する日）。

| 項目 | 値 |
|---|---|
| 入所日時 | **空欄のまま**（初日に入力済みのため） |
| 退所予定日時 | **入力する**（帰る予定の日時） |
| 連泊チェック | **ON**（チェックを入れる） |

![連泊最終日の入力例](images/00_form_overnight_last.png)

> **ポイント**：入所日時を入れてしまうと、システムが「新しい単泊」と判定してダブりカウントになります。**必ず空欄**にしてください。

## 連泊パターンの早見表

| 種類 | 入所日時 | 退所予定日時 | 連泊チェック |
|---|---|---|---|
| 単泊 | ○入力 | ○入力 | OFF |
| 連泊・初日 | ○入力 | **空欄** | **ON** |
| 連泊・中日 | **空欄** | **空欄** | **ON** |
| 連泊・最終日 | **空欄** | ○入力 | **ON** |

> **覚え方**：連泊は「入所日時は**初日だけ**」「退所予定日時は**最終日だけ**」「連泊チェックは**全日ON**」。

![連泊チェックの位置](images/00_form_checkbox.png)

## 食事・服薬の入力ルール（2026-04 仕様）

行政様式（実績記録票）に合わせて、食事と服薬は**時間帯ごとに分かれて**います。

| 項目 | 入れるタイミング |
|---|---|
| 夕食 | 夕食を提供したとき |
| 朝食 | 朝食を提供したとき |
| 昼食 | 昼食を提供したとき（**他サービス併給時は「-」**） |
| 服薬(夜) | 夜の服薬があったとき |
| 服薬(朝) | 朝の服薬があったとき |

> **該当時間帯がない場合**：その項目は「-」を選んでください。空欄でも問題ありませんが、「-」のほうが「該当なし」が明確です。

---

## 睡眠まわりの入力ルール（2026-04 仕様）

旧「睡眠」(○/×) は廃止し、**3項目すべて必須プルダウン**で入力します。

| 項目 | 選択肢 | 入れるタイミング |
|---|---|---|
| 入眠時刻 | 20:30 / 20:40 / 20:50 / 21:00 / 21:10 / 21:20 / 21:30 / 21:40 / 21:50 / 22:00 | 児童が入眠した時刻に最も近い10分刻みを選ぶ |
| 朝4時チェック | 睡眠 / 覚醒確認後に付き添い | 4時の見回り時の状態を選ぶ |
| 起床時刻 | 6:00 / 6:10 / 6:20 / 6:30 / 6:40 / 6:50 / 7:00 / 7:10 / 7:20 / 7:30 | 児童が起きた時刻に最も近い10分刻みを選ぶ |

> **特例（林夏渚さん）**：入眠時刻は **`22:00`** を選んでください（コード上の特例扱いは廃止し、運用ルールで揃えます）。

## よくあるトラブル

| 症状 | 原因と対処 |
|---|---|
| 連泊なのに月別集計で1日しかカウントされない | 連泊チェックがOFFになっている／退所予定日時を初日に入れてしまった可能性。Webビュー（修正ツール）で修正してください |
| 児童名のプルダウンに名前がない | 児童マスタに未登録、または「停止」状態の可能性。管理者に [10_児童を追加する.md](10_児童を追加する.md) で確認を依頼 |
| スタッフ名のプルダウンに自分の名前がない | スタッフマスタに未登録の可能性。管理者に [11_スタッフを追加する.md](11_スタッフを追加する.md) で追加を依頼 |
| 間違えて送信してしまった | 直接スプレッドシートを書き換えず、Webビュー（修正ツール）で修正・削除してください |

---

## 大事なお願い

- **送信は翌朝に行ってください**。フォーム送信日が「利用日」として扱われます（記録日項目は廃止されました）。
- 連泊の入退所日時ルールを守らないと、月別集計・利用枠カウントがズレます。
- 送信した内容を直す場合は、**スプレッドシートを直接編集しない**でください（Webビューで修正）。
