# 振り分けロジック 仕様書

## 概要

月間利用枠に対して実来館数が少ない児童に「振り分け」来館日を自動割当てし、確定来館記録シートに「データ区分=振り分け」として書き込む。

実装: `gas/allocation.gs`

---

## 実行タイミング

| 方法 | 関数名 | タイミング |
|------|-------|---------|
| 自動（月初） | `runMonthlyProcessAutomatic` | 翌月1日 AM6時（前日の属する月=前月が対象） |
| 手動（メニュー） | `runMonthlyProcess` → 月選択 | 「来館管理」メニュー → 月次一括処理 |
| 振り分けのみ手動 | `runAllocationManual` | 月別集計シートのB1/B2で年月を指定して実行 |

既に振り分け済みの月を手動再実行すると確認ダイアログが表示される（既存振り分けデータが全上書きされる旨を警告）。

---

## 処理フロー（`allocateRemainingPoints_`）

```
Step 0: 確定来館記録を対象月の実データで最新化（updateConfirmedVisits）
Step 1: 振り分け対象児童を取得
Step 2: 実来館数・来館日マップを算出
Step 3: 残枠のある児童を抽出し、優先度順にソート
Step 4: 既存振り分け行を削除（洗い替え）
Step 5: 対象月の全日付を取得
Step 6: 日別来館数マップを作成（実データのみ）
Step 7: 補完データ（スタッフ名・入退所時刻・生活記録）を事前計算
Step 8: 年間利用枠チェック用の年累計を取得
Step 9: 児童ごとに振り分けを実行
Step 10: 確定来館記録シートに書き込み（既存データとマージして日付順ソート）
Step 11: 7人満枠の日にスタッフ2を補完
Step 12: 月別集計を更新
```

## 振り分け対象の児童

| 入所状況 | 退所状況 | 対象 |
|---------|---------|------|
| 稼働 | — | ○ |
| 休止 | — | × |
| 退所 | 別施設移動無 | ○（退所月の残枠のみ） |
| 退所 | 別施設移動 | × |

### 実来館2日未満ガード

対象月の実来館が **2日未満** の児童は入所状況にかかわらず振り分け対象外とする（退所フラグ付け忘れ等で実態不在の児童へ満額振り分けされる誤動作を防ぐため）。
当月の実来館が 0〜1 件の児童はログに「対象月実来館N日（2日未満のため振り分け対象外）」を出力し、`childrenWithRemaining` から除外される。

## 振り分け数の決定

### 基本計算

```
月間残枠 = 月間利用枠 ± delta - 実来館数
delta = Math.floor(Math.random() * 3) - 1  // -1, 0, +1 のいずれか
```

### 年間利用枠がある場合の上限調整

```
年間残枠 = 年間利用枠 - 当年の確定来館記録（実データ+振り分け）の累計来館数
月次上限 = min(月間枠±delta, 年間枠の平日按分)
最終振り分け数 = min(月次上限 - 実来館数, 年間残枠)
```

#### 年間枠の平日按分計算（`calcMonthlyQuota_`）

```
月次利用枠 = round(年間利用枠 × 該当月の平日数 / 年間の総平日数)
```

平日 = 月〜金（祝日考慮なし）。月ごとの平日数差によりばらつきが生まれ、12ヶ月合計は年間枠と近似するが必ずしも一致しない。

## 候補日の選定ルール

各日付に対して以下を順番にチェックし、すべて通過した日のみ候補とする。

| チェック | 除外条件 |
|---------|---------|
| 1 | 既に該当児童が来館済みの日 |
| 2 | その日の新着者数（= 入所日がその日の宿泊数）が MAX_VISITS_PER_DAY（= 7名）以上の日 |
| 3 | 設定シートの営業日以外の曜日（営業日が空欄の場合は全曜日対象） |
| 4 | 児童マスタの「非来館曜日」に指定された曜日 |

> **満枠判定の基準**: 新着者数は **利用日 == 入所日** となる行（=その日が入所初日のレコード）のみをカウントする。複数日宿泊で持ち越されている展開行は新着者扱いしない。月間利用枠の消費は展開後の各日付でカウント（各日1枠消費）する。
> **利用日の取得**: 実データ側のレコード参照は `getRowRecordDate_()` を介して RECORD_DATE 列または TIMESTAMP 日付（記録日廃止後はこちらが主）を取得する。

通過した日を2プールに分ける:
- **preferredDates**: 来館曜日（児童マスタ）に該当する日
- **otherDates**: それ以外の営業日

来館曜日プールを優先して埋め、足りなければ otherDates で補う。

### 均等分散

各プール内では、その時点で **日別来館数が最も少ない日から順** に選択する（sort → shift）。他の児童の振り分けで日別カウントが更新されるため、後から処理される児童ほど空いている日を選びやすくなる。

## 補完データの算出（`computeChildDefaults_`）

振り分け行に書き込む体温・食事・入浴等は以下の優先順で決定する。
2026-04 から食事・服薬は分離項目として個別に最頻値算出する。
さらに 2026-04（後半）で旧「睡眠」を **入眠時刻 / 朝4時チェック / 起床時刻** の3項目に分割し、それぞれに **加重ランダム値**（同月の実データ分布を確率重みとして抽選）を採用する。

| 項目 | 優先1 | 優先2 |
|------|-------|-------|
| スタッフ1 | 設定シート row18「固定スタッフ」（曜日別スタッフ運用に対応） | — |
| スタッフ2 | 空欄（7人満枠日のみ後補完） | — |
| 入所・退所時刻 | 同月の自児童実データの最頻値 | 設定シート row3/row4→`ALLOCATION_DEFAULTS` |
| 体温 | 同月の自児童実データの最頻値（小数1桁丸め） | 設定シート row6→36.5 |
| 夕食 / 朝食 / 昼食 | 同月の自児童実データの最頻値（各列独立） | 設定シート row7/row8/row9 →`ALLOCATION_DEFAULTS`（昼食は既定「−」で他サービス併給想定） |
| 入浴 | 同月の自児童実データの最頻値 | 設定シート row10→○ |
| 入眠時刻 | 同月の自児童実データの**加重ランダム**（出現頻度を確率重みに、20:30〜22:00 / 10分刻みから抽選） | 設定シート row11→`21:00` |
| 朝4時チェック | 同月の自児童実データの**加重ランダム**（睡眠 / 覚醒確認後に付き添い） | 設定シート row12→`睡眠` |
| 起床時刻 | 同月の自児童実データの**加重ランダム**（出現頻度を確率重みに、6:00〜7:30 / 10分刻みから抽選） | 設定シート row13→`7:00` |
| 便 | 同月の自児童実データの最頻値 | 設定シート row14→○ |
| 服薬(夜) / 服薬(朝) | 同月の自児童実データの最頻値（各列独立） | 設定シート row16/row15→○ |
| その他連絡事項 | 自児童のノートからランダム | 全児童のノートからランダム→設定シート row17→「特になし」 |

`ALLOCATION_DEFAULTS`（`utils.gs`）のデフォルト値:
- 入所: `17:00`, 退所: `08:00`（翌日朝扱い）, 体温: `36.5`
- 夕食: `○`, 朝食: `○`, 昼食: `−`（他サービス併給の標準想定）
- 入浴: `○`
- 入眠時刻: `21:00`, 朝4時チェック: `睡眠`, 起床時刻: `7:00`
- 便: `○`
- 服薬(夜): `○`, 服薬(朝): `○`
- 連絡事項: `特になし`

> **設定シート側**: `SETTINGS_ROW` は食事（夕/朝/昼）・睡眠3項目（入眠時刻/朝4時チェック/起床時刻）・服薬（夜/朝）ともに行単位で分離済み。`getAllocationDefaultsFromSettings_` は各行の値をそのまま該当項目に割り当てる。

### 退所時刻が入所時刻以前の場合

退所日時が入所日時と同日かつ時刻が前の場合（例: 入所17:00 → 退所08:00）、退所日時を翌日に+1日する。
**この+1日処理は時刻のみフォールバック時のみ適用**。フル日時で入退所が来ているレコードはそのまま使用する（2026-04 修正）。

## 宿泊期間の展開（フォーム1行=1宿泊方式）

実データ（フォーム回答）の集計は、フォーム1行ごとに `expandStayToDates_(checkIn, checkOut)` で入所日〜退所予定日の各日付に展開して扱う。

- `visitCountMap` / `visitDateMap`: 展開後の各日付でカウント
- `dailyVisitCounts`: 入所日のみカウント（新着者数判定用）
- 月またぎ宿泊（例: 3/31 入所 → 4/1 退所予定）も日付ごとに展開される

詳細は `docs/specs/google-form.md` の「宿泊レコード設計」を参照。

## 7人満枠日のスタッフ2補完（`fillStaff2ForFullDays_`）

振り分け書き込み後、対象月の確定来館記録を走査し、**その日の新着者数（= 利用日 == 入所日 となる行）が MAX_VISITS_PER_DAY（= 7名）以上** の日で、スタッフ2が空欄のレコード（利用日がその満枠日と一致する行すべて）に、設定シート row18 の固定スタッフ名（初期値: 溝口母）を書き込む。

複数日宿泊の持ち越し展開行は満枠カウントには含めないが、親の入所日が満枠であればスタッフ2は補完される。

## 優先度ソート

```
区分1 < 区分2 < ... < 区分5 < 未設定（9999）
```

`parsePriority_`: 「区分N」→ N, 数値文字列 → 数値, それ以外 → 9999
