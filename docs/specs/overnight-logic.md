# 宿泊期間の日付展開 仕様書

## 概要

フォーム1行に入所日時・退所予定日時の両方を必ず記入する運用（フォーム1行=1宿泊）。確定来館記録への書き込み時は、入所日〜退所予定日を1日ずつ展開して各日付ごとに1行を生成する。月別集計・年間利用枠のカウントは展開後の各日付（=日数）で行う。

---

## 日付展開ロジック（`expandStayToDates_`）

実装: `utils.gs`

### 動作

```
入力: recordDate（記録日）, checkIn（入所日時）, checkOut（退所日時）
出力: 各宿泊日のDate配列
```

#### 日数の算出

| 条件 | 処理 |
|------|------|
| checkIn と checkOut が両方フル日時（1900年以降） | 日付差 + 1 日として展開 |
| どちらかが時刻のみ（1899/12/30基準） | 1日として扱う |
| checkIn が不正値 | recordDate から1日 |

#### 展開の起点

展開は **recordDate（記録日）** を起点として i=0, 1, 2... と加算する。

```
例: 3/1入所 → 3/3退所（2泊3日）
→ recordDate = 3/1
→ 展開後: [3/1, 3/2, 3/3]（3行）
```

## 月またぎカウントルール

### フォーム回答の月別取得（`getFormResponsesByMonth`）

実装: `utils.gs`

入所日時〜退所日時の**滞在期間**が対象月と重なるレコードを取得する。

```
monthStart = new Date(year, month-1, 1)
monthEnd   = new Date(year, month, 0)  // 月末日

条件: stayStart <= monthEnd AND stayEnd >= monthStart
```

これにより、例えば「2/28入所 → 3/2退所」のレコードは2月集計にも3月集計にも含まれる。

### カウント時の月フィルタ

展開後の各日付が **対象月に属する日のみ** カウントする（`countVisitsByScope_` 内の `matchesScope_`）。

```
例: 2/28〜3/2（2泊3日）
  → 展開後: [2/28, 3/1, 3/2]
  → 2月集計: 2/28の1日カウント
  → 3月集計: 3/1, 3/2の2日カウント
```

## 確定来館記録への書き込みルール

実装: `confirmed-visits.gs`

| 列 | 値の扱い |
|----|---------|
| データ区分（A列） | 「実データ」固定 |
| 利用日（B列） | 展開後の各日付（例: 3/1, 3/2, 3/3） |
| 入所日時（F列） | フォーム入力の入所日時を全展開行に保持 |
| 退所予定日時（G列） | フォーム入力の退所予定日時を全展開行に保持 |
| その他列 | 元フォームの値をすべての行で共有 |

> 入所日時・退所予定日時は全展開行に同じ値が入る（どの行が「入所日」「退所日」かは利用日と入所日時/退所予定日時の比較で判別する）。

---

## 月指定あり・なしの挙動差

`updateConfirmedVisits(year, month)` の引数有無で処理が変わる。

| 呼び出し方 | 処理 |
|----------|------|
| 年月指定あり | 対象月の実データ行のみ洗い替え。他月の実データ・振り分け行は保持 |
| 年月指定なし | 全期間の実データを洗い替え。振り分け行は保持 |

振り分けロジック（`allocation.gs`）は内部で `updateConfirmedVisits(year, month)` を呼び出してから振り分けを実行する。
