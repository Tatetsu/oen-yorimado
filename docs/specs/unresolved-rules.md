# 未実装ルール 仕様書

行政様式の実績記録票や運用上の業務ルールとして存在するが、現行 GAS では未実装のロジックを集約する。現在 Python 側 (`dummy-data/scripts/`) のみで処理されていることの注記を兼ねる。

最終更新: 2026-04-23

## 運用方針（2026-04-23 確定）

§1〜§3（他サービス併給・重度支援加算・送迎記録表）は **GAS 側での実装予定なし**。これらは過去分ダミーデータの生成のための補助ロジックであり、本番運用では不要と判断された。`dummy-data/scripts/` の Python スクリプトで CSV 出力するのみとする。

§4（食事・服薬単一値の旧データ移行）は 2026-04 以前の本番データが存在する場合に限り必要。新規運用開始（2026-05〜）のみであれば対応不要。

## 1. 他サービス併給

### 業務ルール
- 同日に別サービスを併用した日は、昼食を提供しない（昼食欄は「−」）
- 重度支援加算の 12:30 枠も「−」となる
- 算定日数・送迎加算のカウントは通常どおり

| Python (`generate_from_pdf.py`) | PDFの「他サービス併給」列から抽出し、昼食=「−」および重度12:30=「−」に反映 (L398-403, L678-684) |

## 2. 重度支援加算

### 業務ルール
- 対象は重度支援区分 (`MASTER_COL.PRIORITY` = 区分1〜5) が設定された児童
- 来所単位でチェック表を作成
- 時間帯列: 17:00〜翌8:00 の毎時 (16列) + 12:30 + 睡眠チェック (21:00 or 22:00, 4:00, 7:00)
- 入所時刻 ≤ 時刻 ≤ 退所時刻 の範囲のみ記入、範囲外は空欄
- 12:30: 退所時刻 ≥ 12:30 なら ○（他サービス併給時は「−」）
- 睡眠: 夜通し滞在時のみ 21:00/4:00/7:00 の各列に ○/× を記入
- 特例: 林夏渚 (受給者証番号 9200809896) は睡眠チェック時刻 21:00 → 22:00

### 現状
| 側 | 実装 |
|----|-----|
| GAS | **定数のみ**。`MASTER_COL.MEDICAL_SUPPORT(7)` と `MASTER_COL.PRIORITY(8)` は定義済、集計ロジックはない |
| Python | `generate_from_pdf.py:636-741` に完全実装。月次CSV `dummy-data/児童別/重度支援加算/*.csv` を出力 |

### 将来の GAS 実装案
1. 新シート「重度支援加算」を追加
2. 関数 `buildSevereSupportRecords(year, month)` で確定来館記録から対象児童のレコードを展開し、時間帯セルを計算
3. 林夏渚の睡眠時刻特例は児童マスタ側に「睡眠チェック時刻」列を追加するか、コード内ハードコードで対応

## 3. 送迎記録表

### 業務ルール (議事録 20260408 / 20260421 準拠)
- 送迎区分: 夕迎え / 朝送り / 昼送り の3種類
- 時間帯
  - 夕迎え: 17:30〜18:40（通常）
  - 夕迎え (特別メンバー含む日): 17:20〜19:00
  - 朝送り: 07:40〜08:40（退所時刻 08:00/08:30 の児童）
  - 昼送り: 12:40〜13:20（退所時刻 12:00 の児童）
- 特別メンバー (時間帯拡張トリガー): 寺田菫、園田柚真、園田葵、富永日向、小野凌来
- 夕迎え除外: 櫻井麻智、櫻井斗麻、櫻井帆斗（月・水）
- 同日・同区分の児童を1行の利用者列に集約、①〜⑳ の丸数字で列挙
- 兄弟グループ (同姓) は並べて配置 (末尾集約ロジック `reorder_transport_records.py`)

### 現状
| 側 | 実装 |
|----|-----|
| GAS | **未実装** |
| Python | `generate_sougei_record.py` / `reorder_transport_records.py` に完全実装。`dummy-data/送迎記録表/*.csv` を出力 |

### 将来の GAS 実装案
1. 新シート「送迎記録表」を追加（ヘッダー: 日付/曜日/送迎区分/送迎職員/介助者/送迎方法/車番/点呼確認/利用者/開始時刻/終了時刻/備考）
2. 関数 `buildTransportRecords(year, month)` で確定来館記録から送迎行を生成
3. 特別メンバー/除外メンバー/兄弟グループは児童マスタの追加列で管理（ハードコード回避）

## 4. 食事・服薬単一値の旧データ移行

### 背景
2026-04 の FORM_COL 分離変更により、以下の旧データが残存している:
- フォーム応答シート: 旧I列「食事」→ 新I〜K列(夕食/朝食/昼食)
- フォーム応答シート: 旧M列「服薬」→ 新O〜P列(服薬(夜)/服薬(朝))

### 現状
`gas/*.gs` の FORM_COL 参照はすべて新スキーマを前提とする。旧データの行を読むと、新列(夕食/朝食/昼食・服薬(夜)/服薬(朝))が空欄扱いになる。

### 必要な対応（本番運用前）
1. スタッフ用_回答シートで旧I列の値を新I列 (夕食) にコピー
2. 旧M列の値を新O列 (服薬(夜)) にコピー
3. 旧J〜L列 (入浴/睡眠/便) と 旧N列 (連絡事項) を新L〜N列および新Q列に再配置
4. Google フォーム本体で、食事→3問・服薬→2問への質問追加
5. 旧質問を削除または「過去運用」表示

マイグレーションスクリプトは未作成。実施時は `gas/migration_2026_04_meal_split.gs` 等で一時関数化する想定。
