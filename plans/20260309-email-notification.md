# F-07: 保護者向け来館報告メール

## 何を
前日のフォーム回答（実データ）に基づき、来館した児童の保護者宛に来館記録メールを翌朝自動送信する。

## なぜ
保護者に児童の施設での様子を日次で共有し、安心感と情報透明性を提供するため。

## どのように

> **2026-04（後半）の改訂**: フォームから「記録日」項目を廃止したことに伴い、自動送信ロジックを「前日の記録日のレコード」→「過去24時間以内に送信されたフォーム回答（タイムスタンプ基準）のうち未送信のもの」に変更。実装は `sendDailyVisitReports` → `sendVisitReportsRecent_` を呼ぶ構成。手動送信は「指定日のタイムスタンプ日付」基準。共通処理は `processEmailSend_` に集約。詳細は `docs/specs/email-notification.md` を参照。

### ファイル構成
- `gas/email.gs` — メール送信ロジック（新規作成）
- `gas/main.gs` — メニュー・トリガー追加（既存変更）
- `gas/utils.gs` — メールテンプレート定数追加（既存変更）

### 処理フロー（2026-04 後半改訂後）
```
1. 過去24時間（now - 24h 〜 now）のレンジを算出
2. スタッフ用_回答シートから「タイムスタンプが範囲内 + EMAIL_SENT 列が空」のレコードを抽出
3. 各レコードに対して:
   a. 児童名から児童マスタを検索
   b. 保護者メールアドレスを取得（空の場合はスキップ）
   c. メールテンプレートに来館データを埋め込み
   d. MailApp.sendEmail で送信
4. 送信結果をLogger.logに記録
```

### メールテンプレート
- スクリプトプロパティ `FACILITY_NAME` で施設名を設定
- スクリプトプロパティ `EMAIL_SENDER_NAME` で送信者表示名を設定
- 件名: `【{施設名}】{M月D日} {児童名}さんの来館記録`
- 本文テンプレートは `EMAIL_TEMPLATE` 定数として `utils.gs` に定義

### テンプレート本文（想定）
```

いつもお世話になっております。{施設名}です。
```

### トリガー
- 毎朝8時に時間トリガーで `sendDailyVisitReports` を実行
- `setupEmailTrigger` 関数でトリガー設定可能
- カスタムメニューに「来館報告メール手動送信」を追加（テスト・リカバリ用）
- 手動実行時はダイアログで対象日付を指定可能（デフォルト: 前日）

### 仕様詳細
- 同日に複数児童が来館した場合: 保護者ごとに個別メール（1児童1通）
- 保護者メールアドレスが空の児童: スキップ（Logger.logに記録）
- フォーム回答がない日: 何も送信しない
- GAS MailApp の日次送信上限（100通/日）は児童数10名以下のため問題なし

### 想定リスク
- 保護者メールアドレスの入力ミス → メール送信エラーをLogger.logに記録、他の児童の送信は継続
- フォーム回答の利用日（タイムスタンプ日付）と実行日のタイムゾーンずれ → Session.getScriptTimeZone() を使用
- 入退所時間のフォーマット不一致 → Utilities.formatDate で統一フォーマット

### 意思決定の理由
- MailApp を使用（GmailApp ではなく）: シンプルな送信のみで十分、権限スコープも最小限
- テンプレートを定数で管理: 本文変更時にコード修正が必要だが、スクリプトプロパティに長文を入れるより保守しやすい
- 個別メール方式: 将来的にまとめメールに変更する可能性があるが、現時点では個別で実装（要件未確定のため）
