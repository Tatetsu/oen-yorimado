# プランファイル: Phase 2 余りポイント振り分け（F-05 / F-06）

## 何を
前月の来館記録から余りポイント（月間利用枠 - 実来館回数）を算出し、未来館日に自動振り分けする機能を実装する。

## なぜ
月間利用枠を消化しきれなかった児童に対し、余った枠を前月の営業日に均等配置することで、実績管理の正確性を担保する。

## どのように

### 新規ファイル
- `gas/allocation.gs` — 振り分けロジック本体（F-05 / F-06）

### 既存ファイル修正
- `gas/monthly-summary.gs` — 振り分け分も来館数に含めるよう修正
- `gas/main.gs` — カスタムメニューに「振り分け実行」追加、月初トリガー設定関数追加

### 振り分けロジック（`allocateRemainingPoints_`）

```
入力: 対象年月（year, month）
1. 入所状況「あり」の児童を取得
2. フォーム回答から対象月の実来館データ取得
3. 各児童の実来館回数を算出
4. 残枠（月間利用枠 - 実来館回数）> 0 の児童を抽出
5. 優先度順にソート（数値が小さい順）
6. 対象月の全日付（1日〜末日）を取得
7. 各日付の既存来館数マップを作成（実データ + 振り分け済み）
8. 振り分け記録シートから対象月の既存データを削除（洗い替え）
9. 児童ごとに:
   a. 来館済み日を除外
   b. 来館曜日に該当する未来館日を優先候補に
   c. 候補日のうち既存来館数 < 7 の日を抽出
   d. 来館数が少ない日から優先配置（均等分散）
   e. 来館曜日候補で足りない場合は他の営業日に配置
   f. 振り分け結果を記録
10. 振り分け記録シートに書き込み
11. 確定来館記録を再生成（updateConfirmedVisits）
12. 月別集計を更新（updateMonthlySummary）
```

### 振り分け理由
- ユーザー指示により「振り分け理由」列は不要。ALLOCATION_COLからREASON列を削除済み。
- 振り分け記録は: 対象年月, 児童名, 振り分け日, 実行日時 の4列構成。

### 来館曜日の扱い
- 現状は1つの曜日のみ（例: "月"）
- カンマ区切り複数曜日にも将来対応可能な実装にする
- 曜日文字列 → 数値変換: 日=0, 月=1, 火=2, 水=3, 木=4, 金=5, 土=6

### 手動実行（F-06）
- 月別集計シートのB1セル（対象年月ドロップダウン）の値を使用
- カスタムメニューから実行

### 月初自動実行（F-05）
- 毎月1日に前月を対象として実行する時間トリガー
- トリガー設定関数を用意（手動で1回実行）

### 月別集計の修正
- `updateMonthlySummary` の来館数に振り分け分を加算

## 制約条件
- 1日あたり最大7名（実データ + 振り分けの合計）
- 同一児童は同一日に重複不可
- 来館曜日の日を優先して振り分け
- 日別来館数ができるだけ均等になるよう分散
- 優先度が高い（数値が小さい）児童から先に振り分け
- 振り分け対象は入所状況「あり」の児童のみ（「なし」は除外）

## 想定リスク
- 全営業日が7名満枠の場合、振り分け先がない → 振り分け不可としてログ出力
- 来館曜日が未設定の児童 → 全営業日をランダム候補として扱う
- 月間利用枠が0または未設定 → 振り分け対象外

## 意思決定の理由
- 洗い替え方式: 現時点では再実行時に前回結果を削除して再生成。将来的には確定済みデータの保護機能を追加予定
- 「稼働」のみ対象: 入所状況「なし」の児童は利用実態がないため振り分け不要
- 均等分散: 特定の日に偏らないよう、来館数が少ない日から優先的に配置
