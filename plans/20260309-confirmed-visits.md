# 確定来館記録シート（F-03）の新規作成

## 何を
要件定義書 3.5節に定義された「確定来館記録」シートを実装する。
フォームの回答（実データ）+ 振り分け記録を統合した最終データシート。
児童別ビュー（F-04）の参照元をこのシートに切り替える。

## なぜ
- 児童別ビューが2シート（フォームの回答 + 振り分け記録）を直接読んでおり、要件定義と乖離
- 確定来館記録を中間レイヤーとして置くことで、参照が1シートに統一され高速化
- Phase 2（振り分け機能）実装時の基盤になる

## どのように

### 変更対象ファイル
1. `gas/utils.gs` — 定数追加（SHEET_NAMES, CONFIRMED_COL, ヘルパー関数）
2. `gas/confirmed-visits.gs` — 新規作成（F-03: 確定来館記録生成）
3. `gas/setup.gs` — 確定来館記録シートのセットアップ追加
4. `gas/child-view.gs` — 参照元を確定来館記録に変更
5. `gas/main.gs` — メニュー・トリガーに確定来館記録更新を追加

### 確定来館記録の列構成（要件定義書準拠）
確定来館記録の最新の列構成は `docs/requirements.md` 3.4節を正とする。本プランでは初期計画時点の概略のみ示す。

| 列 | 項目 | 備考 |
|----|------|------|
| A | データ区分 | 実データ / 振り分け |
| B | 利用日 | 旧名: 記録日（2026-04 後半でラベル変更）。入所日〜退所予定日を1日ずつ展開 |
| C | スタッフ1 | 振り分けは固定スタッフ |
| D | スタッフ2 | 任意。満枠日は固定スタッフで補完 |
| E | 児童名 | |
| F | 入所日時 | フォーム入力値を全展開行に保持 |
| G | 退所予定日時 | フォーム入力値を全展開行に保持 |
| H | 送迎(往) | 利用日==入所日 → 1 |
| I | 送迎(復) | 利用日==退所予定日 → 1 |
| J | 体温 | 振り分けは補完値 |
| K-M | 夕食/朝食/昼食 | 振り分けは補完値 |
| N | 入浴 | 振り分けは補完値 |
| O | 入眠時刻 | 振り分けは加重ランダム値 |

### 更新タイミング
- F-02（月別集計）実行後に自動呼び出し
- フォーム送信トリガー経由でも更新
- メニューから手動実行可能

### 意思決定の理由
- 全月分を一括保持する方式（月別フィルタではなく全データ保持）
  - 理由: データ量が少なく（最大10児童×31日×12ヶ月）、全月保持でもパフォーマンス問題なし
  - 児童別ビューでの月フィルタは読み取り時に行う
