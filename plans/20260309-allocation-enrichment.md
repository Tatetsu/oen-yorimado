# プラン: 振り分け記録の補完データ永続化

## 日付: 2026-03-09

## 何を
振り分け記録シートに補完データ（スタッフ名・入退所時間・体温・生活記録・連絡事項）を永続化し、
確定来館記録→児童別ビューで振り分け行にも実データ相当の情報が表示されるようにする。

## なぜ
- 現状、振り分け行は全項目が空欄で児童別ビューの情報が欠落している
- 都度計算だとランダム選択や最頻値が変わり、後日見直した時に異なる表示になる
- 振り分け実行時に1回だけ計算・保存することで再現性を保証する

## どのように

### 1. 振り分け記録シートの列拡張（utils.gs）

ALLOCATION_COL を拡張:
```
A: 対象年月
B: 児童名
C: 振り分け日
D: スタッフ名（児童マスタの担当スタッフ）
E: 入所時間（最頻値 / デフォルト 09:00）
F: 退所時間（最頻値 / デフォルト 17:00）
G: 体温（最頻値 / デフォルト 36.5）
H: 食事（最頻値 / デフォルト ○）
I: 入浴（最頻値 / デフォルト ○）
J: 入眠時刻（加重ランダム / デフォルト 21:00）
K: 便（最頻値 / デフォルト ○）
L: 服薬（最頻値 / デフォルト ○）
M: その他連絡事項（自児童ノートからランダム→他児童→デフォルト「特になし」）
N: 実行日時
```

### 2. 補完値計算ロジック（allocation.gs に追加）

- `computeChildDefaults_(childName, formResponses, masterData)` を新設
- 同月の実データから児童別に最頻値を算出
- ノートは自児童 → 他児童 → デフォルトの優先順でランダム選択
- 実データなし時はデフォルト値を使用

### 3. 振り分け結果の書き込み拡張（allocation.gs）

- `allocationResults` 配列を14列に拡張
- `writeAllocations_()` の書き込み列数を14に変更

### 4. 確定来館記録生成の修正（confirmed-visits.gs）

- 振り分けレコードから補完済みデータを読み取る（空欄ではなく）
- ALLOCATION_COL の新列を参照

### 5. セットアップの更新（setup.gs）

- 振り分け記録シートのヘッダー・列幅を拡張

### 6. 要件定義書の更新（docs/requirements.md）

- 振り分け記録シートの列定義を更新
- 確定来館記録の「振り分けの場合は空欄」→「振り分け実行時に補完」に変更

## 想定リスク
- 既存の振り分け記録データが4列しかないため、拡張後に再振り分けが必要
  → 既存データは手動で再実行してもらうか、古いデータは空欄のまま許容
- ランダム選択のノートがシード固定でないため、再振り分け時は変わる
  → 振り分け記録に保存済みなので、再実行しない限り変わらない（許容範囲）

## 意思決定の理由
- 別シート追加ではなく振り分け記録の拡張を選択：データフローをシンプルに保つため
- 補完値を振り分け実行時に永続化：再現性を保証するため（都度計算だと結果が変わる）
- 最頻値を採用（平均ではなく）：体温36.45等の不自然な値を避けるため
